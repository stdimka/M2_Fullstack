Методы работы с атрибутами HTML элементов

1. getAttribute(name) — получение значения атрибута


let elem = document.getElementById("myElement");
console.log(elem.getAttribute("class")); // Получает значение атрибута class

2. setAttribute(name, value) — установка или изменение атрибута


elem.setAttribute("class", "new-class");
elem.setAttribute("data-info", "example"); // Добавляет кастомный data-атрибут

3. removeAttribute(name) — удаление атрибута


elem.removeAttribute("class"); // Удаляет атрибут class

4. hasAttribute(name) — проверка существования атрибута


console.log(elem.hasAttribute("class")); // true или false

5. Прямой доступ к свойствам DOM

Некоторые атрибуты доступны как свойства объекта элемента: 


console.log(elem.id); // Получает id элемента
elem.id = "newId"; // Изменяет id элемента

console.log(elem.href); // Получает ссылку <a>
elem.href = "https://example.com"; // Изменяет ссылку

Но важно помнить, что не все атрибуты доступны через свойства.

6. Работа с dataset (data-атрибуты)

Если у элемента есть data-* атрибуты, их можно получать и устанавливать через dataset: 


console.log(elem.dataset.info); // Получает data-info
elem.dataset.info = "newValue"; // Устанавливает data-info="newValue"

7. Методы classList для работы с классами

Методы работы с атрибутами и методы работы с классами (classList) связаны, но не одно и то же. Если цель — управлять class, то лучше использовать classList, потому что это удобнее и безопаснее, чем setAttribute("class", ...).

element.classList.add(className) — добавляет класс


elem.classList.add("highlight");

element.classList.remove(className) — удаляет класс


elem.classList.remove("highlight");

element.classList.toggle(className [, force]) — переключает класс 


elem.classList.toggle("highlight"); // Добавляет, если нет, удаляет, если есть
elem.classList.toggle("highlight", true); // Принудительно добавляет
elem.classList.toggle("highlight", false); // Принудительно удаляет

element.classList.contains(className) — проверяет наличие класс 


console.log(elem.classList.contains("highlight")); // true или false

element.classList.replace(oldClass, newClass) — заменяет один класс другим 


elem.classList.replace("old-class", "new-class");

Почему classList лучше setAttribute("class", ...)?

  • Безопаснее — не перезаписывает все классы, а изменяет нужный.
  • Проще — не нужно вручную разбирать строку class.
  • Быстрее — не требует полного обновления DOM-атрибута.

Если основная задача — работа с атрибутами, classList можно не добавлять. Но если ваш код часто изменяет классы, их стоит включить в описание, потому что это лучший способ управления class.
