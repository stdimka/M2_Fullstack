Работа со свойствами и методами объектов в JavaScript (CRUD)

1. Доступ к существующим свойствам и методам

Можно использовать:

  • Точечную нотацию: object.property
  • Квадратные скобки: object["property"]
const user = { name: "Alice", age: 30 };
console.log(user.name); // Alice
console.log(user["age"]); // 30

2. Добавление новых свойств и объектов

Свойства можно добавлять динамически:

user.email = "alice@example.com";
user["isAdmin"] = true;
console.log(user);

3. Изменение существующих свойств

user.age = 31;
console.log(user.age); // 31

4. Удаление свойств

delete user.email;
console.log(user.email); // undefined

Прототипное наследование

const animal = { eats: true };
const rabbit = Object.create(animal);
rabbit.jumps = true;
console.log(rabbit.eats); // true

Изменение прототипа

animal.walk = function () { return "Walking..."; };
console.log(rabbit.walk()); // "Walking..."

Вывод

  • Свойства можно добавлять, изменять, читать и удалять динамически.
  • Объекты могут наследовать свойства от других объектов.
  • Изменения в прототипе влияют на всех наследников.

Таким образом, изменение свойств и методов объектов можно подразделить на два типа:

1. Изменение локального объекта

Это когда свойство или метод добавляется или изменяется непосредственно в самом объекте. Эти изменения касаются только этого конкретного объекта, и они не затрагивают прототип.

Пример:


const animal = { eats: true };
const rabbit = Object.create(animal);

rabbit.eats = false; // Это локальное изменение для rabbit, не влияет на animal

console.log(rabbit.eats); // false (локальное свойство)
console.log(animal.eats); // true (прототип не изменён)

2. Изменение прототипа

Когда мы меняем свойство или метод в прототипе объекта, эти изменения отражаются на всех объектах, которые наследуют от этого прототипа. Прототип — это как "шаблон" для всех экземпляров.

Изменения, сделанные в прототипе, влияют на все объекты, наследующие от этого прототипа, даже если в них нет такого свойства.

Пример:


const animal = { eats: true };
const rabbit = Object.create(animal);

animal.eats = false; // Это изменение в прототипе

console.log(rabbit.eats); // false (поскольку rabbit наследует от animal)

Как различить?

Локальные изменения — касаются только конкретного объекта, который изменяется (например, rabbit.eats = false;).

Изменения прототипа — касаются всех объектов, которые наследуют этот прототип (например, изменение animal.eats = false; затронет все объекты, наследующие от animal).

Это два различных способа управления свойствами объектов в JavaScript, и важно понимать, как они работают, чтобы корректно использовать прототипы и локальные данные.
