Разница между null и undefined в JavaScript

Тип null представляет намеренное отсутствие значения. Это значение часто используется для инициализации переменных, которые будут позже присвоены объектом, или для очистки значения переменной.

В отличии от этого, тип undefined представляет значение переменной, которая была объявлена, но не инициализирована. Это значение также возвращается при доступе к несуществующему свойству объекта или же при обращению к элементу массива, по не существующему индексу.

Сходство и отличие Python None от JavaScript null и undefined

В Python None ближе всего к null в JavaScript, так как оба означают "отсутствие значения". Однако есть несколько отличий:

  • None в Python имеет тип NoneType, а null в JavaScript – это object.
  • undefined в JavaScript обозначает переменную, которой не присвоено значение. В Python такой ситуации нет — если переменная не определена, возникнет ошибка NameError.
  • Если функция в JavaScript не возвращает значение, она возвращает undefined. В Python функция без return возвращает None.
